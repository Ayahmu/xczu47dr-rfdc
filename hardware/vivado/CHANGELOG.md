# Vivado Build System - Changelog

## 2026-05-12 - ILA Removal and Build Script Improvement

### Issue
Block Design TCL script contained ILA (Integrated Logic Analyzer) debug IP cores that caused nested sub-design errors when recreating the project from TCL:
```
ERROR: [Vivado 12-3563] The Nested sub-design can only be generated by its parent sub-design.
```

This error occurred because:
1. ILA IP cores (`system_ila_0` and `system_ila_1`) generate nested Block Designs
2. SmartConnect and DDR4 IPs also have similar nested design issues
3. TCL scripts exported from Vivado may not properly recreate these nested designs

### Changes Made

#### 1. Removed ILA Debug IP from Block Design TCL
**File**: `bd/design_1.tcl`

Removed the following ILA instances and their connections:
- `system_ila_0` - 6-slot ILA monitoring AXIS interfaces
- `system_ila_1` - 1-slot ILA monitoring AXI memory-mapped interface

**Lines removed**:
- ILA IP check from `list_check_ips` (line 136)
- `system_ila_0` instantiation and configuration (lines 764-802)
- `system_ila_1` instantiation and configuration (lines 848-865)
- ILA AXIS interface connections (lines 877, 879, 881, 891-893)
- ILA AXI interface connections (line 887)
- ILA clock connections (lines 905, 949)
- ILA reset connections (lines 929, 939)

**Backup created**: `bd/design_1.tcl.backup`

#### 2. Improved Build Script to Use Existing Projects
**File**: `build.sh`

Modified Step 2 to check for existing project files before attempting to recreate:

**Before**:
```bash
# Step 2: Create Vivado Project
print_step "Step 2/5: Creating Vivado project..."
vivado -mode batch -source scripts/create_project.tcl -notrace
```

**After**:
```bash
# Step 2: Create Vivado Project (or use existing)
PROJECT_FILE="${WORK_DIR}/${PROJECT_NAME}.xpr"
if [ -f "${PROJECT_FILE}" ]; then
    print_step "Step 2/5: Using existing Vivado project..."
    print_info "Found existing project: ${PROJECT_FILE}"
    print_info "To recreate project from scratch, use --clean option"
else
    print_step "Step 2/5: Creating Vivado project..."
    vivado -mode batch -source scripts/create_project.tcl -notrace
    if [ $? -ne 0 ]; then
        print_error "Project creation failed"
        exit 1
    fi
    print_info "Project created successfully"
fi
```

### Benefits

1. **Avoids Nested Sub-design Errors**: 
   - Once a project is successfully created, subsequent builds reuse it
   - No need to recreate Block Design from TCL every time
   - Avoids ILA, SmartConnect, and DDR4 nested design issues

2. **Faster Build Times**:
   - Skips project creation step (~30-40 seconds saved)
   - Only recreates when explicitly requested with `--clean`

3. **More Robust Workflow**:
   - Works around Vivado TCL export/import limitations
   - Preserves manually made changes in Vivado GUI
   - Clear message to user about using existing project

### Usage

**Normal build (uses existing project if available)**:
```bash
./build.sh --skip-synth --skip-impl --skip-bitstream
```

**Force recreation from TCL**:
```bash
./build.sh --clean --skip-synth --skip-impl --skip-bitstream
```

**Full build with existing project**:
```bash
./build.sh
```

### Impact on Debug Capability

**Removed**:
- ILA debug probes for AXIS data streams (ADC/DAC data)
- ILA debug probes for AXI memory-mapped transactions

**Workaround if debugging is needed**:
1. Open project in Vivado GUI: `vivado work/zcu216_rfdc.xpr`
2. Manually add ILA IP cores as needed
3. Configure debug probes
4. Run synthesis/implementation from GUI
5. Use Hardware Manager for debugging

**Note**: For production builds, ILA debug IP is typically not needed and removing it:
- Reduces FPGA resource usage
- Simplifies the design
- Avoids TCL script issues

### Recommendations

1. **For development**: Use the existing project workflow (current default)
2. **For clean builds**: Use `--clean` flag to recreate from scratch
3. **For debugging**: Add ILA manually in Vivado GUI when needed
4. **For CI/CD**: Consider using `--clean` to ensure reproducible builds

### Known Limitations

1. **TCL Script Limitations**: 
   - The `bd/design_1.tcl` script may still fail if used to create a project from scratch
   - SmartConnect and DDR4 nested designs may cause similar errors
   - Recommended to use existing project or regenerate TCL from GUI

2. **Manual Changes**: 
   - Any manual changes made in Vivado GUI will be preserved
   - These changes won't be reflected in the TCL script
   - Use `File → Export → Export Block Design` to update TCL if needed
